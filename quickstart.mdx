## Prerequisites

Before starting, ensure you have:

<Callout kind="info">

- A modern web browser like Chrome `{version >= 100}` or Firefox `{version >= 90}`
- An email address for account verification
- Optional: A Google account for faster sign-up

Artist Game works best on desktop for initial setup, though mobile support is available.

</Callout>

## Sign Up and Log In

Create your Artist Game account in under 2 minutes.

<Tabs>
  <Tab title="Email Sign-Up" icon="mail">
    <Steps>
      <Step title="Visit Dashboard" icon="globe">
        Navigate to `https://dashboard.example.com` and click **Sign Up**.
      </Step>
      <Step title="Enter Details">
        Provide your email, create a strong password `{>= 12 characters}`, and agree to terms.
      </Step>
      <Step title="Verify Email">
        Check your inbox for the verification link from `no-reply@artistgame.com` and click it.
      </Step>
      <Step title="Log In">
        Return to the dashboard, enter your credentials, and access your workspace.
      </Step>
    </Steps>
  </Tab>
  <Tab title="Google Sign-Up" icon="apple">
    <Steps>
      <Step title="Connect Google">
        On `https://dashboard.example.com`, select **Continue with Google**.
      </Step>
      <Step title="Authorize">
        Grant Artist Game permission to access your basic profile and email.
      </Step>
      <Step title="Complete Setup">
        Customize your workspace name and start using Artist Game.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## Initial Workspace Setup

After logging in, configure your default workspace.

<Steps>
  <Step title="Name Your Workspace" icon="edit-3">
    Click the workspace dropdown in the top-left and select **New Workspace**. Enter a name like `My Project Docs`.
  </Step>
  <Step title="Set Brand Color">
    Go to **Settings > Branding** and input your brand color `#3B82F6`.
  </Step>
  <Step title="Invite Team Members" icon="users">
    In **Team** settings, add emails. They receive invites via `https://dashboard.example.com/invite/{token}`.
  </Step>
</Steps>

<Callout kind="tip">

Save your workspace ID `{e.g., ws_abc123}` from the settings pageâ€”you'll need it for API integrations.

</Callout>

## Create Your First Project

Build your first documentation space programmatically or via UI.

<Tabs>
  <Tab title="UI Method" icon="mouse-pointer">
    <Steps>
      <Step title="New Project">
        From the sidebar, click **+ New Project** and name it `Artist Game Quickstart`.
      </Step>
      <Step title="Add First Page">
        Select **Add Page**, title it `Introduction`, and start editing with Markdown or MDX.
      </Step>
    </Steps>
  </Tab>
  <Tab title="API Method" icon="code">
    Use the Artist Game API to create projects instantly.

    <CodeGroup tabs="cURL,JavaScript">
      ```bash
      curl -X POST https://api.example.com/v1/projects \
        -H "Authorization: Bearer YOUR_API_KEY" \
        -H "Content-Type: application/json" \
        -d '{
          "name": "API Quickstart Project",
          "workspaceId": "ws_abc123"
        }'
      ```
      ```javascript
      const response = await fetch('https://api.example.com/v1/projects', {
        method: 'POST',
        headers: {
          'Authorization': 'Bearer YOUR_API_KEY',
          'Content-Type': 'application/json',
        },
        body: JSON.stringify({
          name: 'API Quickstart Project',
          workspaceId: 'ws_abc123',
        }),
      });
      const project = await response.json();
      console.log(project.id);
      ```
    </CodeGroup>

    <ParamField path="name" param-type="string" required="true">
      Project display name.
    </ParamField>

    <ParamField header="Authorization" param-type="string" required="true">
      Bearer token from dashboard settings.
    </ParamField>
  </Tab>
</Tabs>
